### 3. Запуск AI run + отмена

**Действия:**

- [ ] Открытие панели Execution
- [ ] Нажатие "Run" для задачи
- [ ] Проверка создания run записи
- [ ] Проверка статуса "running"
- [ ] Отмена через кнопку "Cancel"
- [ ] Проверка статуса "cancelled"

**Ожидаемый результат:**

- Run создается в БД с task_id
- Статус меняется: queued → running
- События task:onEvent отправляются в UI
- Отмена обновляет статус
- Run события записываются в run_events

- UI подписывается на события через IPC
- События приходят в реальном времени
