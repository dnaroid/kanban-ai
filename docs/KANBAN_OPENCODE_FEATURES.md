# Kanban for Headless OpenCode + oh-my-opencode — описание фич

## Цель продукта
Десктопное Electron‑приложение “канбан + AI‑исполнители”, где каждая таска может:
- иметь свой чат/лог выполнения,
- выполняться агентами через headless OpenCode (с плагином oh-my-opencode),
- вести работу в отдельной git‑ветке,
- после приёмки создавать PR и мержиться (включая AI‑помощь в конфликт-резолве),
- поддерживать планирование (таймлайн), приоритеты, теги, зависимости, метрики.

## Основные сущности
- **Workspace**: набор проектов и общие настройки (провайдеры git, модели, права, секреты).
- **Project**: репозиторий + доски + правила (DoD, merge gates, политики AI).
- **Board**: колонки/свимлейны, WIP‑лимиты, правила переходов.
- **Task**: юзер‑стори/баг/спайк/техдолг, поля, статус, чат, логи, ветки, PR, артефакты.
- **Agent Role**: BA/Dev/QA/Architect/PM/Security и пользовательские роли.
- **Run/Job**: отдельный запуск агента (план, кодинг, тестирование, резолв конфликтов).
- **Artifacts**: диффы, файлы, отчёты тестов, ADR, чеклисты, ссылки.

## 1) Канбан и управление работой

### Доски и колонки
- Drag & drop карточек между колонками.
- Кастомные колонки/статусы, правила переходов (state machine).
- **WIP‑лимиты** по колонкам/свимлейнам.
- “Blocked” как свойство таски: причина, дата, кто заблокировал.

### Swimlanes
Опциональные дорожки:
- по эпикам / релизам / исполнителям / типам / приоритетам.

### Зависимости
- Связи: **blocks / blocked by / relates to / duplicates**.
- Визуализация зависимостей: мини‑граф, предупреждения о циклах.
- Автопроверка: “нельзя завершить таску, если она блокируется”.

### Подзадачи и чеклисты
- Subtasks с прогрессом.
- Чеклисты: DoD, QA, релизный чеклист.
- Автоматические чеклисты по шаблонам.

### Поля и кастомизация
- Базовые поля: заголовок, описание, приоритет, теги, тип, оценка, дедлайн, исполнитель.
- **Acceptance Criteria** (AC) отдельным полем, **Definition of Done** (DoD) отдельным полем.
- Кастомные поля (типизированные): enum/число/дата/пользователь/URL/markdown.

### Поиск и представления
- Полнотекстовый поиск (по таскам, комментариям, логам).
- Фильтры/сортировка/группировки.
- Сохранённые представления (Saved Views) + быстрые команды (Cmd+K).

## 2) Планирование: таймлайн, релизы, загрузка

### Таймлайн (Gantt/Timeline)
- Перетаскивание дат, масштабирование.
- Отображение зависимостей и критического пути (если включено).
- Milestones / релизы / версии.

### Спринты и capacity
- Спринты/итерации, velocity, capacity (часы/points).
- Диагностика перегруза: подсветка по лимитам.

---

## 3) Git, ветки, PR, CI — как часть статуса таски

### Branch-per-task workflow
- На старт работы: auto‑branch (на основе шаблона имени).
- Привязка: task ↔ branch ↔ commits ↔ PR.
- Авто‑черновик PR (draft) опционально.

### PR/Review flow
- Автогенерация PR title/description (из story + AC + диффа).
- Labels/reviewers/CODEOWNERS.
- Merge strategies: merge/squash/rebase (настройка проекта).

### Merge gates (правила “нельзя в Done пока…”)
Примеры условий:
- CI зелёный,
- минимум N approvals,
- тест‑план заполнен,
- есть обновление документации,
- нет открытых зависимостей/блокировок.

### CI статусы и артефакты
- Статус пайплайна в карточке таски.
- Просмотр логов/артефактов (ссылка или импорт).

## 4) AI‑слой: роли, операции, контроль

### Ролевые агенты
- Предустановки: **BA, Dev, QA, Architect, PM, Security**.
- Пользовательские роли и “наборы навыков” (skill packs).
- Handoff: последовательность ролей (BA → Architect → Dev → QA).

### Операции (кнопки‑действия, а не “просто чат”)
- **BA**: сформировать user story + AC + edge cases + вопросы.
- **Architect**: ADR/RFC, варианты, trade‑offs, риски.
- **Dev**: план изменений (файлы/модули), оценка, реализация, рефакторинг.
- **QA**: тест‑план, negative cases, regression checklist, автотесты.
- **Security**: threat‑model, секреты, зависимости, SAST рекомендации.
- **Merge**: AI‑резолв конфликтов (предложить/применить после подтверждения).

### Контекст‑пакеты
- Для каждой таски хранится “контекст‑пакет”: ссылки, файлы, выдержки, решения.
- Версионирование контекста (snapshot), чтобы можно было воспроизвести run.

### Контроль стоимости и ограничений
- Бюджеты: токены/время/параллелизм на таску/проект.
- Режимы: plan-only, code-only, critique-only.
- Политики доступа: allowlist директорий, запрет чтения секретов, redaction логов.

## 5) Окно “чат таски” и наблюдаемость

### Два потока
- **Discussion**: человеческое обсуждение.
- **Execution Log**: что реально делал агент (команды, ответы, ошибки, решения).

### Дифф и артефакты
- Встроенный diff viewer с комментами.
- Список изменённых файлов, связка с коммитами.
- Артефакты: тест‑репорты, ADR, чеклисты, ссылки, файлы.

### Replay
- Воспроизведение run: шаги, команды, применённые патчи.

## 6) Коллаборация и UX

### Комментарии и уведомления
- Комментарии, @mentions, реакции.
- Уведомления: desktop + “тихий режим”.

### Быстрые действия
- Cmd+K (создать таску, назначить роль, запустить run, открыть PR).
- Массовые операции (теги/приоритет/перенос/архив).

### Настройки и профили
- Профили моделей/провайдеров на проект.
- Темы, хоткеи, форматирование.

---

## 7) Оффлайн и хранение

### Offline-first
- Локальная БД (SQLite) + миграции + бэкапы.
- Очередь синхронизации (если включены внешние интеграции).

### Безопасность
- Хранилище секретов (через OS keychain).
- Логи с редактированием чувствительных данных.
- Политики “safe mode” для shell.

---

## 8) Интеграции и расширяемость

### Провайдеры
- Git: GitHub/GitLab/Bitbucket (минимум GitHub/GitLab в MVP).
- Webhooks для обновления статусов PR/CI.
- Опционально: синхронизация с Jira/GitHub Issues.

### Плагины
- Плагины UI (рендер карточек/панелей) и backend (валидаторы DoD, источники задач, AI‑операции).

---

## 9) Метрики и качество

- Lead time / Cycle time / Throughput.
- Aging WIP (стареющие карточки).
- Доля blocked, флаппинг статусов.
- “AI impact”: сколько run’ов, стоимость, доля откатов/повторных попыток.

---

## MVP vs “позже”
### MVP (первые версии)
- Проекты + доска + таски + drag/drop + теги/приоритет.
- Чат таски + запуск роли (BA/Dev/QA) через headless OpenCode.
- Branch-per-task + draft PR + статусы PR/CI.
- Простые merge gates + ручной merge.
- Локальная SQLite, keychain для токенов.

### Позже
- Полный Gantt с критическим путём, capacity/velocity.
- Плагины, двухсторонняя синхра с Jira.
- Расширенный replay, анализ метрик и прогнозы.
