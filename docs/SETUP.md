# Запуск OpenCode сервера

> **Примечание**: Проект использует Git Worktrees для параллельной разработки. Каждый worktree — это отдельная директория с собственной веткой. См. [GIT_WORKTREES.md](GIT_WORKTREES.md) для деталей.

## Архитектура (Способ 1: Подключение к запущенному серверу)

Приложение работает **как клиент** и подключается к уже запущенному OpenCode серверу.

```
┌─────────────────────────────────────────────────────────┐
│  Приложение kanban-ai (Клиент)              │
│                                                   │
│  ┌─ OpenCodeSessionManager ──────────────────┐  │
│  │                                            │  │
│  │  createOpencodeClient({                   │  │
│  │    baseUrl: OPENCODE_URL                   │  │
│  │  })                                      │  │
│  │                                            │  │
│  │  client.session.create()                   │  │
│  │  client.session.prompt()                    │  │
│  └──────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                      ↓ HTTP
┌─────────────────────────────────────────────────────────┐
│  OpenCode Server (Запускается отдельно)          │
│                                                   │
│  • opencode serve --port 4096                    │
│  • Управляет сессиями AI-агентов                │
│  • Сохраняет историю чата                           │
│                                                   │
│  URL: http://localhost:4096 (по умолчанию)  │
└─────────────────────────────────────────────────────────┘
```

## Требование

Для работы с AI-агентами через SDK необходимо запустить **OpenCode сервер отдельно**.

## Запуск сервера (Ручной режим)

### Шаг 1: Запустите OpenCode сервер в отдельном терминале

```bash
# Глобальная установка (если ещё не установлен)
npm install -g @opencode-ai/cli

# Запуск сервера
opencode serve --port 4096

# Или из исходников
cd /Volumes/128GBSSD/Projects/opencode
pnpm run serve --port 4096
```

### Шаг 2: Запустите приложение kanban-ai

```bash
cd /Volumes/128GBSSD/Projects/kanban-ai
pnpm dev
```

### Шаг 3: Проверьте подключение

```bash
# Проверка health endpoint
curl http://localhost:4096/health

# Ожидаемый ответ:
# {"status":"ok"}
```

## Настройка URL

Скопируйте `.env.example` в `.env` и настройте URL сервера:

```bash
cp .env.example .env
```

В `.env`:

```env
VITE_API_URL=http://localhost:3000

# OpenCode SDK
# URL запущенного сервера OpenCode serve
OPENCODE_URL=http://localhost:4096
```

Если сервер запущен на другом порту, измените `OPENCODE_URL`:

```env
OPENCODE_URL=http://localhost:4100
```

## Решение проблем

### "fetch failed"

Эта ошибка означает, что SDK не может подключиться к OpenCode серверу.

**Причины:**

1. Сервер не запущен
2. Неверный порт в `.env`
3. Другой URL сервера

**Решение:**

```bash
# 1. Проверьте запущен ли сервер
ps aux | grep "opencode.*serve"

# 2. Проверьте порт
lsof -i :4096

# 3. Если сервер не запущен - запустите в отдельном терминале
opencode serve --port 4096

# 4. Или измените порт в .env и запустите сервер на другом порту
```

## Режим разработки с автоматическим запуском

Для автоматического запуска сервера при старте приложения можно добавить в `package.json`:

```json
{
  "scripts": {
    "dev": "concurrently \"opencode serve --port 4096\" \"vite\""
  }
}
```

И установить concurrently:

```bash
pnpm add -D concurrently
```

## Преимущества Способа 1

1. **Сервер запускается один раз** - экономия ресурсов
2. **Множество приложений** могут подключаться к одному инстансу
3. **Сессии сохраняются** между перезапусками клиента
4. **Независимость от процесса** - можно управлять сервером отдельно
